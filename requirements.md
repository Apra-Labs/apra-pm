# Requirements: AGY provider-agnostic /auto-sprint skill

Epic: apra-pm-lmx

## Goal

Add `skills/auto-sprint/` to the `apra-pm` repo and update `install.mjs` so that
`node install.mjs --llm agy` deploys the full skill to
`~/.gemini/antigravity-cli/skills/auto-sprint/`.

## Constraint 1: identical input JSON to auto-sprint.js

The skill and runner MUST accept exactly the same input grammar as
`.claude/workflows/auto-sprint.js`. Read the `meta` block at the top of
`auto-sprint.js` for the canonical field list. The four invocation forms must work:

  /auto-sprint BD-7
  /auto-sprint BD-1 BD-2
  /auto-sprint ["BD-1","BD-2"]
  /auto-sprint {"issues":["BD-7"],"branch":"feat/x","goal":"P1"}

## Constraint 2: same inner logic as auto-sprint.js

Keep runner.js inner workings as close to auto-sprint.js as possible:
- Same phase order: Plan -> Develop -> Test -> Harvest
- Same agent roster: planner, plan-reviewer, doer, reviewer, deployer,
  integ-test-runner, ci-watcher, harvester
- Same schema names: REVIEW_SCHEMA, PLAN_REVIEW_SCHEMA, DOER_STATUS_SCHEMA,
  HARVEST_SCHEMA, CI_SCHEMA, INTEG_RUN_SCHEMA
- Same deploy.md + integ-test-playbook.md detection for Test phase gating
- Same pure-JS cost/calibration arithmetic:
    fitStreakToContext, truncateStreakToCeiling, computeSprintQuote,
    computeSprintAnalysis, computeUpdatedCalibration, buildSprintSummary
  Load these from cost.js (extracted by install.mjs from auto-sprint.js).
- Same no-progress abort (if prevOpenIds == currentOpenIds after cycle N>1 -> abort)
- Same log() format: prefixed [RUNNER] with ISO timestamp
- Same dispatchLedger structure and cost summary table at the end

## Constraint 3: deterministic routing (zero LLM orchestration tokens)

The orchestrator loop MUST be pure Node.js. No LLM token spent on:
- deciding which task to run next (use local bd execSync)
- evaluating verdicts (pure string comparison)
- exit/continue decisions (pure JS conditionals)
- beads reads (local execSync)

Structured output enforcement: every execute_prompt that expects JSON must append
a RESPOND WITH ONLY VALID JSON schema block and retry up to 3 times on parse failure.

## Constraint 4: provider-agnostic fleet dispatch

The runner dispatches via the apra-fleet MCP client. Members are pre-registered
by the user (one-time) following member-setup.md. The runner selects members by
name (pm-planner, pm-reviewer, pm-doer-cheap, pm-doer-std, pm-doer-premium,
pm-harvester). Tier->model resolution is fleet server-side. The runner NEVER
hardcodes model IDs.

## Constraint 5: install.mjs integration

- Existing --llm claude install is unchanged
- --llm agy MUST deploy skills/auto-sprint/ to the AGY skill dir as step [5/5]
- --uninstall --llm agy MUST remove the auto-sprint skill dir
- Post-install message for agy MUST show the 4 invocation forms and name the USP

## Deliverables

1. `skills/auto-sprint/SKILL.md` - skill entry point
2. `skills/auto-sprint/runner.js` - deterministic Node.js orchestrator (~450 lines)
3. `skills/auto-sprint/member-setup.md` - one-time fleet member registration guide
4. `install.mjs` - updated (AGY deploy step, agyOnlyPermissions, uninstall, post-msg)

## Reference files (do not modify)

- `.claude/workflows/auto-sprint.js` - canonical reference for logic and schemas
- `skills/pm/cost.md` + `skills/pm/cost.js` (generated by install.mjs) - cost fns
- `install.mjs` (already partially edited - review current state before planning)
- `agents/planner.md`, `agents/doer.md`, etc. - agent runbooks (do not modify)
